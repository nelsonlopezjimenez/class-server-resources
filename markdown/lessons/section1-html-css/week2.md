## What is HTML?

**HTML** stands for **HyperText Markup Language**. Let's break that down:

- **HyperText**: Text that contains links to other text. This is what makes the web "web-like" - documents connected to other documents
- **Markup**: A system of annotating text to indicate how it should be structured or displayed
- **Language**: A set of rules and syntax for creating documents

**HTML is NOT a programming language** - it's a markup language. It doesn't have logic, loops, or conditions. Instead, it describes the structure and content of web pages.

### A Brief History of HTML

**1991**: Tim Berners-Lee creates HTML at CERN (European Organization for Nuclear Research) in Switzerland. The first version had only 18 tags.

**1995**: HTML 2.0 becomes the first standard specification.

**1997**: HTML 3.2 adds support for tables, applets, and text flow around images.

**1999**: HTML 4.01 introduces better support for stylesheets and scripting.

**2014**: HTML5 becomes the official standard, adding semantic elements, multimedia support, and modern web features.

**Today**: HTML5 is the current standard, continuously evolving with new features.

### HTML's Impact on the World

HTML revolutionized communication and information sharing:
- **Made information accessible**: Anyone can publish content to billions of people
- **Created the digital economy**: E-commerce, social media, streaming services all built on HTML
- **Democratized publishing**: No need for printing presses or distribution networks
- **Connected the world**: Information flows instantly across continents
- **Enabled innovation**: Billions of websites, apps, and services built on this foundation

## HTML vs Markdown vs XML

### What is Markdown?

**Markdown** is a lightweight markup language created in 2004 by John Gruber. It's designed to be easy to read and write, and can be converted to HTML.

**Markdown Example:**

# This is a heading

This is a paragraph with **bold text** and *italic text*.

- Item 1
- Item 2
- Item 3

### Why Markdown is Better for Readers:

Plain text readability: You can read markdown files as-is without converting them
Minimal syntax: Uses intuitive symbols like # for headings, * for emphasis
Quick to write: Much faster than typing HTML tags
Clean and simple: No opening/closing tags cluttering the text
Popular for documentation: Used in GitHub, Reddit, Discord, and many note-taking apps

### When to Use Markdown:

1. Documentation and README files
1. Note-taking and writing
1. Blog posts (many platforms convert markdown to HTML)
1. Quick formatting without complexity

### What is XML?
XML (Extensible Markup Language) is a markup language designed to store and transport data. It's similar to HTML but focuses on data structure rather than display.
XML Example:

```xml
<?xml version="1.0"?>
<person>
  <name>Jane Doe</name>
  <age>28</age>
  <email>jane@example.com</email>
</person>
```
### Key Differences:

#### HTML: 
Designed for displaying content in web browsers
XML: Designed for storing and transporting data
Markdown: Designed for easy writing and reading

#### XML Characteristics:

Stricter rules than HTML
Custom tag names (you define your own tags)
Used for data exchange between systems
Must be well-formed (properly nested and closed)

#### When to Use XML:

Configuration files
Data exchange between different systems
RSS feeds
Storing structured data

#### Comparison Table
| Feature | HTML | Markdown | XML |
|---------|------|----------|-----|
| Purpose | Display web pages | Easy writing/reading | Store/transport data |
| Readability | Medium (tags visible) | High (plain text) | Low (verbose) |
| Learning Curve | Easy | Very Easy | Medium |
| Flexibility | Predefined tags | Limited syntax | Custom tags |
| Browser Support | Native | Needs conversion | Needs parser |

### Day 1: HTML Basics and Document Structure

**Understanding HTML Syntax:**

HTML uses **tags** to mark up content. Tags are keywords enclosed in angle brackets.

```html
<tagname>Content goes here</tagname>
```

**Key Concepts:**

1. **Opening Tag**: `<tagname>` - Marks the beginning of an element
2. **Closing Tag**: `</tagname>` - Marks the end (note the forward slash)
3. **Content**: What appears between the tags
4. **Element**: The complete package (opening tag + content + closing tag)

**Self-Closing Tags:**
Some tags don't need closing tags because they don't contain content:
```html
<br>  <!-- Line break -->
<img src="photo.jpg">  <!-- Image -->
<hr>  <!-- Horizontal rule/line -->
<input type="text">  <!-- Form input -->
```

**HTML Comments:**
Comments are notes in your code that browsers ignore:
```html
<!-- This is a comment. It won't appear on the webpage -->
<!-- Use comments to explain your code or temporarily disable sections -->
```

**The Basic HTML Document Structure:**

Every HTML document follows this template:

```html
<!DOCTYPE html>
<!-- This declaration tells the browser this is an HTML5 document -->

<html lang="en">
<!-- The root element. Everything goes inside this. lang="en" specifies English -->

  <head>
    <!-- The head contains metadata (information about the page) -->
    <!-- Content in the head is NOT displayed on the page -->
    
    <meta charset="UTF-8">
    <!-- Tells browser to use UTF-8 character encoding (supports all languages) -->
    
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <!-- Makes your site responsive on mobile devices -->
    
    <title>My First Webpage</title>
    <!-- The title appears in the browser tab -->
    
  </head>
  
  <body>
    <!-- The body contains all visible content -->
    <!-- Everything you see on a webpage goes here -->
    
    <h1>Hello, World!</h1>
    <!-- This is a heading - the largest and most important -->
    
    <p>This is my first paragraph.</p>
    <!-- This is a paragraph of text -->
    
  </body>
  
</html>
```

**HTML Versions - A Quick Overview:**

- **HTML 1.0 (1991)**: Basic tags for headings, paragraphs, links
- **HTML 2.0 (1995)**: Added forms, tables
- **HTML 3.2 (1997)**: Added styling capabilities
- **HTML 4.01 (1999)**: Better separation of content and style
- **XHTML (2000)**: Stricter version of HTML
- **HTML5 (2014-present)**: Current standard with modern features

**HTML5 Major Features:**

1. **Semantic Elements**: Tags that describe their meaning (`<header>`, `<nav>`, `<article>`, `<footer>`)
2. **Multimedia Support**: Native `<video>` and `<audio>` tags (no plugins needed!)
3. **Graphics**: `<canvas>` for drawing, `<svg>` for scalable graphics
4. **Form Enhancements**: New input types like email, date, number
5. **Local Storage**: Store data in the browser
6. **Geolocation**: Access user's location (with permission)
7. **Offline Support**: Websites can work without internet connection

### Day 2: Text Formatting and Semantic HTML

**Heading Tags:**

HTML has six levels of headings, from most important (h1) to least important (h6):

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Heading Examples</title>
</head>
<body>

  <h1>This is a Heading 1 - Main Title</h1>
  <!-- Use only ONE h1 per page - it's your main title -->
  
  <h2>This is a Heading 2 - Major Section</h2>
  <!-- Use h2 for major sections of your content -->
  
  <h3>This is a Heading 3 - Subsection</h3>
  <!-- Use h3 for subsections within h2 sections -->
  
  <h4>This is a Heading 4</h4>
  <h5>This is a Heading 5</h5>
  <h6>This is a Heading 6 - Smallest Heading</h6>
  <!-- h4-h6 are progressively smaller and less important -->

</body>
</html>
```

**Best Practices for Headings:**
- Use headings in order (don't skip levels)
- h1 is the page title
- Use headings to create an outline of your content
- Don't use headings just to make text big - use them for structure

**Text Formatting Tags:**

```html
<!-- BOLD AND STRONG -->
<b>This text is bold</b>
<!-- b tag makes text bold (visual only) -->

<strong>This text is strong</strong>
<!-- strong tag indicates importance (semantic meaning) -->
<!-- Screen readers emphasize strong text -->
<!-- Prefer <strong> over <b> for meaningful emphasis -->


<!-- ITALIC AND EMPHASIS -->
<i>This text is italic</i>
<!-- i tag makes text italic (visual only) -->

<em>This text is emphasized</em>
<!-- em tag indicates emphasis (semantic meaning) -->
<!-- Prefer <em> over <i> for emphasis -->


<!-- OTHER TEXT FORMATTING -->
<u>Underlined text</u>
<!-- Underlines text (use sparingly - looks like links) -->

<s>Strikethrough text</s>
<!-- Shows text with a line through it (for deleted content) -->

<mark>Highlighted text</mark>
<!-- Highlights text with a yellow background -->

<small>Smaller text</small>
<!-- Makes text smaller - for fine print -->

<sub>Subscript text</sub>
<!-- Subscript - like Hâ‚‚O -->

<sup>Superscript text</sup>
```

**Semantic HTML5 Elements:**

Semantic elements clearly describe their meaning to both browser and developer:

```html
<header>
  <!-- Contains introductory content, logo, navigation -->
  <h1>My Website</h1>
</header>

<nav>
  <!-- Contains navigation links -->
  <a href="#home">Home</a>
  <a href="#about">About</a>
</nav>

<main>
  <!-- Contains the main content of the page -->
  <!-- Use only ONE main element per page -->
  
  <article>
    <!-- Contains independent, self-contained content -->
    <!-- Like a blog post, news article, or forum post -->
    <h2>Article Title</h2>
    <p>Article content goes here...</p>
  </article>
  
  <section>
    <!-- Groups related content together -->
    <!-- Should usually have a heading -->
    <h2>Section Title</h2>
    <p>Section content...</p>
  </section>
  
</main>

<aside>
  <!-- Contains content tangentially related to main content -->
  <!-- Like a sidebar, related links, or ads -->
  <h3>Related Information</h3>
  <p>Side content...</p>
</aside>

<footer>
  <!-- Contains footer information -->
  <!-- Copyright, contact info, links -->
  <p>&copy; 2024 Your Name</p>
</footer>
```

**Why Use Semantic HTML?**
1. **Accessibility**: Screen readers understand page structure better
2. **SEO**: Search engines better understand your content
3. **Maintainability**: Code is easier to read and understand
4. **Future-proof**: Works better with new technologies

### Day 3: Lists, Links, and Navigation

**Lists in HTML:**

There are three types of lists:

**1. Unordered Lists (Bullet Points):**

```html
<ul>
  <!-- ul = Unordered List -->
  <!-- Creates a bulleted list -->
  
  <li>First item</li>
  <!-- li = List Item -->
  
  <li>Second item</li>
  <li>Third item</li>
  
</ul>

<!-- Practical example: -->
<h3>My Skills</h3>
<ul>
  <li>HTML</li>
  <li>Problem Solving</li>
  <li>Communication</li>
  <li>Team Collaboration</li>
</ul>
```

**2. Ordered Lists (Numbered):**

```html
<ol>
  <!-- ol = Ordered List -->
  <!-- Creates a numbered list -->
  
  <li>First step</li>
  <li>Second step</li>
  <li>Third step</li>
  
</ol>

<!-- Practical example: -->
<h3>My Work Experience</h3>
<ol>
  <li>Software Developer at Tech Corp (2022-2024)</li>
  <li>Junior Developer at StartUp Inc (2020-2022)</li>
  <li>Intern at Digital Agency (2019-2020)</li>
</ol>

<!-- You can change the numbering style: -->
<ol type="A">  <!-- A, B, C... -->
  <li>Option A</li>
  <li>Option B</li>
</ol>

<ol type="I">  <!-- Roman numerals: I, II, III... -->
  <li>Chapter One</li>
  <li>Chapter Two</li>
</ol>

<ol start="5">  <!-- Start at 5 instead of 1 -->
  <li>Fifth item</li>
  <li>Sixth item</li>
</ol>
```

**3. Description Lists (Term and Definition):**

```html
<dl>

  <!-- dd = Description Definition -->

  <dd>Cascading Style Sheets - the styling of web pages</dd>
  
  <dt>JavaScript</dt>
  <dd>Programming language that adds interactivity to web pages</dd>
  
</dl>
```
